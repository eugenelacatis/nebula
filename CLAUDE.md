# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Nebula** is a 3D music particle visualizer built with React, Three.js, and the Web Audio API. It renders a real-time particle galaxy that reacts to music frequency analysis, with a procedural nebula skybox whose appearance is dynamically generated by Claude AI based on the music genre/mood.

## Setup

Create `.env.local` in the project root:
```
VITE_ANTHROPIC_API_KEY=your_key_here
```

## Commands

```bash
npm run dev       # Start Vite dev server
npm run build     # Production build
npm run preview   # Preview production build
```

No test suite exists in this project.

## Architecture

The app has a clear separation between React (UI/state), Three.js (rendering), and Web Audio API (audio analysis/synthesis).

### Data Flow

```
React <App /> ──state changes──▶ NebulaApp.js (orchestrator)
                                      │
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                   ▼
             AudioEngine.js    SynthEngine.js    ParticleSystem.js
             (FFT analysis,    (procedural        (GLSL shaders,
              beat detection)   preset tracks)     18k particles)
                    │
                    └──frequency data──▶ NebulaSkybox.js
                                              │
                                        claudeService.js
                                        (Claude Haiku API)
```

### Key Modules

- **`src/NebulaApp.js`** — Central orchestrator. Initializes Three.js scene, `EffectComposer` with bloom post-processing, and runs the main `_animate()` loop. All React-to-3D interactions go through its public API (`togglePlay`, `selectPreset`, `setVolume`, `rebuildParticles`, `loadUploadedFile`).

- **`src/AudioEngine.js`** — Web Audio API wrapper. Performs FFT analysis and extracts bass/mid/high frequency bands. Beat detection uses a rolling bass average with exponential cooldown.

- **`src/SynthEngine.js`** — Purely procedural synthesizer. Three preset tracks (Cosmic Pulse 128BPM, Nebula Drift ambient, Solar Winds 170BPM DnB) built entirely from Web Audio primitives — no audio files needed.

- **`src/ParticleSystem.js`** — Three.js `Points` with custom GLSL vertex/fragment shaders. 18k particles in a 3-arm spiral galaxy distribution. Vertex shader responds to bass/mid/high uniform values and beat flag for pulse/explosion effects. Fragment shader renders soft additive-blended points.

- **`src/NebulaSkybox.js`** — Procedural Perlin noise sphere (inverted geometry) with domain warping. Accepts JSON parameter objects and smoothly lerps to new values. Calls `claudeService.fetchSkyboxParams()` on track changes.

- **`src/claudeService.js`** — Calls Claude Haiku (`claude-haiku-4-5-20251001`) with a detailed prompt to generate skybox color/style parameters JSON. The Vite dev server proxies `/api/claude/*` to `https://api.anthropic.com` to avoid CORS and keep the API key server-side.

### React Layer

- **`src/App.jsx`** — Owns all UI state. Instantiates `NebulaApp` via `useEffect` on mount, calls its methods on state changes. VU bar DOM updates happen directly every frame (bypassing React state for performance).

- **`src/components/UI.jsx`** — Purely presentational. Preset selector, play/pause, volume/bloom sliders, VU bars, particle count toggle, file upload.

## Key Implementation Details

- **AudioContext lazy init**: Browser security requires AudioContext to be created on user gesture. `NebulaApp.togglePlay()` initializes it on first play.
- **Claude API proxy**: The Vite proxy in `vite.config.js` rewrites `/api/claude/*` → Anthropic API. The API key is read from `import.meta.env.VITE_ANTHROPIC_API_KEY` and passed in request headers.
- **Skybox updates**: Fire-and-forget on track select — Claude API calls don't block UI. Parameters lerp smoothly (~1.5s transitions) when new values arrive.
- **Bloom**: `EffectComposer` with `UnrealBloomPass`. Bloom strength dynamically scales with bass frequency every frame.
- **Particle counts**: 3 presets (8k/18k/35k). Changing count disposes old geometry and rebuilds — `rebuildParticles()` in NebulaApp.
- **`src/main.js`**: Old vanilla JS entry point, superseded by `src/main.jsx`. Ignore it.
- **`.next/`**: Leftover from a previous Next.js scaffold, not used.
